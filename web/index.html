About

Kusanagi is a collection of tools intended for trouble shooting including both user space and kernel space tools.
Granted that there are many very useful tools applicable for our trouble shooting activities, but often those tools are inconvenient, have restrictions, difficult to understand their output or so. From another point of view, often we want to have a collected knowledge base to exploit those convenient tools.
Here is a collection of tools and knowhow document intended for those needs. Tools here would help you to analyze system behavior, catch evidences for your guesses or reproduce troubles etc.

Resources

Here is a brief list of what we have now.
  • Tools
    • User Space Tools
      • opanalyze -- an oprofile log file analyzer
        opanalyze is an oprofile log analyzer by which we can detect discontinuous or not smooth changes in sample counts of symbols and understand the system behavior.
      • btx -- a better back trace for crash written in LDAS
        btx is an LDAS script of Alicia, a front-end of crash utility, providing better stack traces.
      • load -- a Simple CPU Load Generator
        Often we need to generate a particular CPU load such as 65% of total CPU power in order to reproduce the trouble. This poor program do that.
    • Kernel Space Tools
      • rye -- an in-kernel event catcher
        rye catches a specific in-kernel event such as a context switch storm and causes a panic.
        Use it with your favorite crash dump utility such as kdump.
      • allie -- a kprobe wrapper for general trouble shooting
        Using allie you can catch calls of a particular in-kernel function and see stack traces (or cause a panic).
      • holden -- a network trouble shooting helper
        holden uses netfilter to catch TCP packets with a specific combination of bits in their TCP headers.
  • Documents
    • Stack Traces of IA32/Linux ([japanese], in preparation)
